# Checklist de Revisión - Results (MDPI)

## A. Enfoque y pertinencia
- [ ] Cada párrafo responde explícitamente a D1, D2, D3 o síntesis transversal.
- [ ] Se eliminaron párrafos que no aportan evidencia nueva.
- [ ] Se removieron frases metadiscursivas repetidas (“en esta sección...”, “como cierre...”) sin valor analítico.

## B. Separación Results vs Discussion
- [ ] No hay recomendaciones de diseño/prescripción en `Results`.
- [ ] Todo contenido de implicaciones/propuesta fue movido a `Discussion`.

## C. Redundancias internas
- [ ] Números PRISMA/retrieval/calidad aparecen una sola vez en texto (el resto remite a figura/tabla).
- [ ] No se repite la misma conclusión en párrafos consecutivos.
- [ ] No se repiten transiciones metodológicas dentro de la misma subsección.

## D. Duplicidad tabla-figura
- [ ] Para cada bloque se definió artefacto principal (tabla o figura).
- [ ] Figuras que replican exactamente conteos de tablas fueron movidas a anexo.
- [ ] Se conservaron figuras de coocurrencia/patrones cuando aportan lectura no visible en tabla.

## E. Citas y ejemplos
- [ ] Se redujo número de citas por idea (2–3 estudios representativos).
- [ ] Listados largos de casos se movieron a anexo si no cambian la conclusión principal.

## F. Coherencia de datos
- [ ] Cifras en texto coinciden con tablas (k/n/%).
- [ ] Se aclaró explícitamente el caso de deduplicación para evitar ambigüedad.
- [ ] Se distinguen claramente “NR” vs “no aplica” vs “ausencia del fenómeno”.

## G. Compactación final
- [ ] Se eliminaron repeticiones de apertura/cierre de subsecciones.
- [ ] Se mantuvo trazabilidad mínima (tabla de trazabilidad por directriz).
- [ ] La narrativa final conserva flujo: evidencia -> patrón -> interpretación.
